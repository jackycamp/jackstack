Path Manager in Python

june 2023 boston

systems around geo-processing tools and spatial algorithms often deal with a significant number of files on disk; and if you tile or distribute parts of said system the number of files grow exponentially.

it is challenging to know which files are inputs or outputs for the pieces in the system. decent documentation doesn't solve this problem either as it is too separate from the code.

some pieces of the system consume the same inputs; the outputs of some pieces may be inputs to some other pieces. as the number of processing modules in a system increases and the number of inputs increases, this becomes messy, frustrating, and inefficient if your team doesn't have a standard.

for some of our spatial data processing pipelines (written in python), my team agreed we should establish a single entry point and solid interface for all files the system might consume or produce.

we came up with a list of requirements for a PathManager:

1. should be able to hit dot and see all of the files for that instance

2. defining a file or path should be trivial

3. needs to support nested directories

4. always takes care of os.path.join for us

5. documenting a path should be easy
